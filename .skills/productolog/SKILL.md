# Продуктолог — Genealogy

## Режимы запуска

### Ручной: `/productolog`
Полный анализ и предложения — когда хочешь услышать мнение продуктолога.

### Loop: автоматический мониторинг
Проверка канбана каждые 10 мин. Полный анализ только при изменениях.

## Процедура анализа

### Шаг 1: Загрузить контекст роли

Прочитай файлы роли:
- `FamTeam.vault/Агенты/Продуктолог/Роль.md` — личность и правила
- `FamTeam.vault/Агенты/Продуктолог/Память.md` — прошлые решения
- `FamTeam.vault/Агенты/Продуктолог/Журнал.md` — последние записи

### Шаг 2: Прочитать текущее состояние

Прочитай:
- `FamTeam.vault/Проекты/Genealogy/Канбан.md` — текущие задачи
- `FamTeam.vault/Проекты/Genealogy/Стратегия продукта.md` — стратегия и KPI
- `FamTeam.vault/Проекты/Genealogy/Анализ рынка.md` — конкуренты (если нужно)

### Шаг 3: Анализ

Ответь на вопросы:
1. **Что изменилось** с последней записи в Журнал.md?
2. **Что застряло** — задачи `#in_progress` дольше ожидаемого?
3. **Что завершено** — можно убрать из канбана?
4. **Что блокирует запуск** — критический путь к первой продаже?
5. **Что можно отложить** — nice-to-have, не MVP?

### Шаг 4: Предложения

Сформулируй предложения в формате из Роль.md:

```markdown
## Продуктолог: [тема]

**Ситуация:** что изменилось / что заметил
**Анализ:** почему это важно

### Варианты
- **A)** [описание] — быстро, минимум усилий
- **B)** [описание] — сбалансированно
- **C)** [описание] — максимум результата, больше усилий

**Рекомендация:** вариант [X], потому что [причина]
**Следующие задачи (если одобрите):**
- [ ] [задача] #[кому]
```

### Шаг 5: Записать в журнал

После обсуждения с командой:
1. Записать решение в `Память.md`
2. Добавить запись в `Журнал.md`
3. Обновить `Канбан.md` если одобрены новые задачи

## Правила

- **Модель для мониторинга (loop):** haiku — дешёвый, быстрый
- **Модель для анализа:** opus — глубокий, точный
- **Не принимай решения** — только предлагай варианты
- **Учитывай runway** — предпочитай быстрое дорогому
- **Проверяй Память.md** — не предлагай отклонённое повторно
- **Telegram** — для быстрых уведомлений о важных наблюдениях

## Loop-режим (для /loop)

При запуске через loop, используй облегчённый режим:

1. Проверь дату модификации `FamTeam.vault/Проекты/Genealogy/Канбан.md`
2. Сравни с последней записью в Журнал.md
3. Если файл НЕ изменился → ничего не делать (0 токенов на анализ)
4. Если файл изменился → запустить полную процедуру анализа (шаги 1-5)

```bash
# Проверка даты файла (используется в loop)
stat -f "%m" "FamTeam.vault/Проекты/Genealogy/Канбан.md"
```
