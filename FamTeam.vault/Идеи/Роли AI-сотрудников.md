# AI-роли (сотрудники) — дизайн системы

## Концепция

AI-роль = постоянный сотрудник с должностью, памятью и инициативой. Не инструмент, а член команды.

## Флоу работы роли

```
Роль просыпается (loop)
    ↓
Проверяет: изменился ли канбан? (0 токенов — проверка даты файла)
    ↓ да
Читает контекст: канбан, история, прошлые решения
    ↓
Оценивает ситуацию: что сделано, что застряло
    ↓
Предлагает следующие шаги (варианты A/B/C)
    ↓
Мы обсуждаем / одобряем / комментируем / отменяем
    ↓
Роль понимает решение → ставит задачи на канбан (мне, Сергею, Юле)
    ↓
Исполнители делают → отмечают выполненным
    ↓
Роль проверяет результат → оценивает → предлагает следующий шаг
    ↓
Цикл повторяется
```

## Умный loop (решение по токенам)

- Каждые 10 минут проверяем ТОЛЬКО дату изменения канбана (0 токенов)
- Если файл изменился → запускаем полный анализ
- Если нет → спим дальше
- Результат: частая проверка без расхода токенов

## Первая роль: Продуктолог

- **Зона:** стратегия, фичи, приоритеты, roadmap
- **Проект:** Genealogy (для начала)
- **Контекст:** анализ рынка, стратегия продукта, канбан, решения
- **Полномочия:** предлагает → мы одобряем → ставит задачи → контролирует

## Будущие роли

- **Маркетолог** — лендинг, SEO, контент, каналы
- **Техлид** — архитектура, код, ревью
- **QA** — тесты, баги, качество
